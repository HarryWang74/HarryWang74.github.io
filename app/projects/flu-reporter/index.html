Project background

Past few years especially in flu seasons, I used to use https://www.google.org/flutrends to check the flu trend in Sydney area. After Google stopped this service on September 2015, I realized similar data still could be found on https://www.google.com/trends/ However, on Google trend page you have to set category, country, region, time … to find the result. That’s a bit annoying.

So I decided to build a small application called “Flu Reporter” and share it with people who still want to use Google flu trend data.

Requirements

Identified requirement for Flu Reporter behavior at beginning

  1. Can send various location information to Google API and read/render the response data
  2. Can be deployed to serve and works as a website
  3. After wrapped by CORDOVA Flu Reporter can be submitted/distributed through application stores

Technologies

The technologies/frame-works used for Flu Reporter

  • Angular single page application
  • Yeoman workflow, use Generator-Jekyllrb as project scaffold
  • Use ngCordova to adapt Cordova plugins
  • Wrap application by Ionic for IOS/Andriod stores release

How Flu Reporter works

To get Google trend data, you need to send Google visualization Query with location specified ISO-3166-2 code to trend API. So Flu Reporter’s application logic is API driven and highly asynchronous.

  1. Flu reporter uses api.geonames.org to convert Geographic coordinate to ISO-3166-2 code
  2. Send Visualization Query with ISO-3166-2 code to Google Trend API
  3. As Trend API only returns data for entirely country, Flu Reporter applies logic to JSON response to find data for specific region
  4. Render data responsively
