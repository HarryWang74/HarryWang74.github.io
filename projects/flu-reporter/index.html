Flu Reporter

Design / Development

Background

In past a few years, I used a Google web service to check the flu trend in Sydney area in flu seasons. After Google stopped this service in September 2015, I realized such data would still be available on Google Trend. However, you have to set up complex search criteria every time when start search which is a bit annoying.

So I decided to build a small application that can automatically grab flu trend data from google API and share it with people who still want to use Google flu trend data.

Application Behavior

  1. Send location information to Google API and render response data
  2. Can be deployed to serve and work as a website
  3. After wrapped by Cordova, application can be submitted through application stores

Built with

  • Flu reporter is an Angular single page application
  • Kickstart with Yeoman workflow, use Generator-Jekyllrb as project scaffold
  • Use ngCordova to adapt Cordova plugins
  • Wrapped by Ionic for IOS/Andriod stores release

  1. Use Geonames API to convert geographic coordinate to ISO-3166-2 code

  3. Apply logic to JSON response to process data for specific region
  4. Render data responsively
